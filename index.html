💾 Respaldo completo (Método principal)
✅ Este es el método correcto. Guarda absolutamente todo: clientes, historial de pagos, archivos adjuntos (en IndexedDB) y mejoras.
El respaldo .json se genera en segundo plano sin bloquear la aplicación, usando procesamiento asíncrono por lotes. Para archivos muy grandes el proceso tarda unos segundos — verás un indicador de progreso.
📊 Exportar / Importar Excel
⚠️ Excel no guarda historial de pagos ni archivos adjuntos. Úsalo para reportes, no como respaldo principal.
🗄️ Almacenamiento — IndexedDB
✅ Windesarrollo Pro usa IndexedDB, que no tiene un tope fijo. El espacio disponible depende del disco duro de tu computador — en Chrome puede usar hasta el 60% del disco libre, en Firefox hasta el 50%. En la práctica dispones de decenas o cientos de GB, muy diferente al límite de 5 MB de LocalStorage.
Jerarquía de datos
Cliente → Proyectos → Historial Mensual → Documentos
Cada cliente puede tener múltiples proyectos. Cada proyecto tiene su propio historial de pagos por mes y sus propios documentos adjuntos separados.
Portabilidad entre dispositivos
IndexedDB es local al navegador, igual que LocalStorage. Para migrar a otro equipo: Guardar respaldo → transferir .json → Cargar respaldo en el nuevo dispositivo.
⚠️ Los datos de IndexedDB también están ligados al navegador y dispositivo. El respaldo .json es tu única copia portable. Hazlo regularmente.
📦 Historial de auto-guardados
El programa guarda automáticamente cada 5 minutos en IndexedDB, con 24 slots rotativos.

💾 Actualización Manual — Guía paso a paso
✅ No requiere conocimientos técnicos. Solo necesitas hablar con Claude en claude.ai.
Pasos
1
Abre claude.ai en tu navegador. Si no tienes cuenta, créala gratis.
2
Escríbele: "Soy usuario de Windesarrollo Pro v4.0.0. Necesito un parche .json para: [describe la mejora]. Genera el JSON completo listo para cargar."
3
Claude responderá con un bloque que empieza con { y termina con }. Cópialo completo.
4
Abre el Bloc de notas (Windows) o TextEdit (Mac), pega el texto y guárdalo como parche.json (extensión .json, no .txt).
5
Vuelve aquí → "Cargar parche .json" → selecciona el archivo → confirma.
🌐 Actualización Online — Guía detallada
Opción A — GitHub (gratis, más confiable)
1
Crea cuenta en github.com.
2
Crea repositorio nuevo → botón verde "New" → nombre: windesarrollo-updates → marca "Public" → "Create repository".
3
"Add file" → "Create new file" → nómbralo updates.json → pega: {"version":"4.0.0","notas":"Sin cambios","patch":""} → "Commit changes".
4
La URL del archivo es: https://raw.githubusercontent.com/TU-USUARIO/windesarrollo-updates/main/updates.json
5
Aquí en el programa → "⚙️ Configurar URL" → pega esa URL → "Guardar y buscar".
6
Cuando Claude te dé un parche: ve a GitHub → abre updates.json → lápiz ✏️ → reemplaza el contenido → "Commit changes". El programa lo descargará automáticamente.
Opción B — Pastebin (más simple)
1
Ve a pastebin.com (sin registro).
2
Pega el parche de Claude → "Syntax Highlighting: JSON" → "Create New Paste".
3
En la URL de resultado, cambia pastebin.com/ por pastebin.com/raw/ → esa es tu URL.
⚠️ La URL debe devolver texto JSON puro. URLs de Google Drive o Dropbox no funcionan directamente.
